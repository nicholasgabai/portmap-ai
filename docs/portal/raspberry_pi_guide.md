# Raspberry Pi Guide

## Purpose

This guide summarizes Raspberry Pi operation for local metadata-first validation.

## What Operators Should Know

Raspberry Pi environments benefit from bounded workloads, compact TUI layouts, and conservative runtime settings. PortMap-AI should prioritize scan limits, metadata summaries, and clear operator review over high-volume processing.

## Key Commands And Concepts

- Validate with focused tests before full-suite runs when resources are constrained.
- Prefer read-only summaries and bounded packet metadata windows.
- Use Raspberry Pi profiles and deployment readiness documentation for local setup.
- Confirm TUI panes do not overflow at the terminal size used by the device.

## Safety Notes

Do not enable privileged packet capture, enforcement, or system-level service changes without an explicit approved workflow.

## Current Limitations

The portal does not include device-specific validation results. Keep real-device notes in private validation records, not public documentation.
