# Open Source / Enterprise Model Guide

## Purpose

This guide explains the current boundary between the local open-source product and future enterprise packaging.

## What Evaluators Should Know

PortMap-AI is currently local-first and metadata-first. Recent commercial-readiness phases added deterministic local models for licensing, control-plane metadata, and customer provisioning. These models prepare later enterprise workflows without implementing hosted billing, customer portals, remote execution, or cloud services.

## Key Concepts

- Open-source core: local visibility, packet metadata intelligence, deterministic AI summaries, governance records, and export-safe models.
- Enterprise readiness: local licensing metadata, deployment records, provisioning models, and future control-plane planning.
- Future SaaS phases should build on local models without weakening privacy, safety, or operator control.

## Safety Notes

Do not treat local commercial-readiness models as online enforcement, billing, authentication, or remote-management systems.

## Current Limitations

No payment processing, customer portal, hosted API, cloud database, SSO provider, or remote license server is implemented by this portal phase.
